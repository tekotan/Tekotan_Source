GreenBall Platform Game on Scratch

Duration: August 30, 2018 - September 9, 2019
Partner: Abhishek Sarepaka Platform: Scratch
Objective: To build a scratch game or animation with user interaction and multiple stages.
Details: For the scratch project, we had to program a scratch game or animation that had user interation along with multiple stages. We chose to create a game because it appealed to our interests. We decided on creating a platform game with realistic gravity effects and movement that agrees with the laws of physics. Since I had previous programming experience, I took the lead in the design of the game, using my experience to teach my partner, while my partner handled the asthetic aspects of the project like designing the levels and characters of the game. Some challenges that arose was correctly modeling the projectile motion of the ball during a jump. We worked around this by researching the ways other people used in their own platform games and modeled our code after theirs. This project taught me how to divide and conquer tasks based on the skills available between the two of us. The documentation for this project can be found here.

Image Manipulation Project

Duration: Febuary 25, 2019 - March 28, 2019
Partner: Andrea Jia Platform: Python
Objective: To develop an image manipulation feature using Python and other APIs
Details: For the image manipulation project we had develop and design an image manipulation feature using python APIs and other modules. The original project provided 3 guided tasks, all involving a client bringing a task. We however decided to develop something more complex, and opt for Task 4, which was the development option. We decided to implement a style transfer system described in the paper A Neural Algorithm of Artistic Style , which would transfer the style of one image onto the content of another image. Since I had prior experience in reading and understanding algorithms in scientific papers, I headed the algorithm implementation while Andrea focused on the user interface and the documentation of the project. Some challenges that arose was in the implementation of the algorithm was that some of the techniques were too exhaustive for the resources available to us, in which case I used prior knowledge of Deep Learning to substitute the techniques with other viable options. Another challenge that arose was figuring out the best way to provide the user with choices in the user interface. We solved this challenge by taking the opinion of our classmates of what they want in a user interface, and using their opinons to build one. This project taught me about how to read and comprehend a scientific paper that outlines an algorithm, and how to use mathematics to understand a computer algorithm. Some soft skills I learned was how to explain an algorithm to someone that has no prior experience on the subject and it's background, something that is a very important skill for the professional world. The documentation for this project can be found here, and for the techinical details and paper summary see my blog on this project here.
